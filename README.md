## Why this notebook is different

The other three notebooks in this series include AI-coached practice scenarios. This one deliberately doesn't. Implementation science and MEL (Monitoring, Evaluation, Learning) frameworks come out of academic peer review, not a two-week corporate training — they reward precision and citation over rehearsal, and a fabricated practice scenario would do the discipline a disservice. This is a cheatsheet for recognising and correctly selecting the right framework, not a sandbox for practicing it.

These frameworks also cluster differently across a project's lifecycle than the other three notebooks. Consulting frameworks cluster early (diagnosing and deciding). These cluster late — almost everything here lives at the *Prove* and *Scale* stages, asking whether something that worked in a trial or pilot survives contact with the real world. See the [notebook hub's Journey map](https://bwtwong.github.io) for how this set slots in against the rest of the series.

## What's included (8 frameworks)

### Determinant Frameworks — what helps or blocks implementation
- CFIR (Consolidated Framework for Implementation Research)
- i-PARIHS (Integrated Promoting Action on Research Implementation in Health Services)
- Normalization Process Theory (NPT)

### Evaluation Frameworks — assessing whether and why it worked
- RE-AIM (Reach, Effectiveness, Adoption, Implementation, Maintenance)
- Realist Evaluation (CMO — Context, Mechanism, Outcome)
- OECD-DAC Evaluation Criteria
- Developmental Evaluation

### Process Frameworks — the cycle of moving evidence into action
- Knowledge-to-Action (KTA) Framework

## A note on sourcing

Every entry here is drawn from the peer-reviewed implementation science and development evaluation literature. This is a practitioner's reference, not a substitute for reading the primary sources — always go to the original paper before applying a framework in real evaluation work.
